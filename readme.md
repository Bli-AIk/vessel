## Introduction

In Rust projects, writing code to parse `.ron` configuration files is an everyday occurrence. **But what about the other way around?**

Asking a human to hand-write thousands of lines of `.ron` files filled with "magic numbers" and precise math (like keyframe animations driven by mathematical formulas, intricate branching dialogue trees, or massive RPG stat tables) is pure torture.

`vessel` was born to flip the script. It allows you to **write code to handle the complex calculations, and automatically generates `.ron` configuration files for you.**

To guarantee safety and to pave the way for supporting multiple languages, the generator code you write is compiled into a WASM Component. `vessel` acts as the "host environment". Its sole job is to run your compiled WASM plugin and save whatever configuration data it spits out to your hard drive.

In short: **Write code to calculate data -> Compile to WASM -> Run with vessel -> Get RON config files.**

## Features

* 🦀 **Code as Data** — Use your favorite programming language to handle complex logic. Say goodbye to manual calculations and copy-pasting configs.
* 🛡️ **Sandbox Safety** — Generator code runs in a completely isolated WASM environment (powered by [Wasmtime](https://wasmtime.dev/)). It is strictly sandboxed and cannot mess up your system.
* 🔌 **Polyglot by Nature** — Any language that can compile to the WASM Component Model can be used to write config generators (Currently, Rust is the primary SDK).
* 🛠️ **Flexible Integration** — Comes with an easy-to-use CLI tool, but can also be embedded directly into your own build scripts as a Rust library.
* 📜 **Auto-Tagging** — Generated RON files automatically get a `// Generated by Vessel` header to prevent confusion with hand-written assets.

## How to Use

Using vessel is a two-step process: first, write a "generator" (WASM side), and then run it with `vessel` (Host side).

### 1. Write a Generator (WASM side)

Since the `vessel` core only provides a standardized WIT contract, you will need to use `wit-bindgen` in your WASM plugin project to generate and implement the interface:

```rust
// src/lib.rs
// Use wit-bindgen to import the vessel interface definition
wit_bindgen::generate!({
    path: "path/to/vessel/wit",
    world: "content-module",
});

struct MyContentPlugin;

// Implement the Guest trait
impl Guest for MyContentPlugin {
    fn build() -> Vec<vessel::build::types::GeneratedFile> {
        // You can use loops, math, and complex logic here
        let my_data = "{ // Configuration data resulting from intense calculations... }";

        vec![vessel::build::types::GeneratedFile {
            path: "my_battle.ron".to_string(), // Tell vessel what to name the file
            ron_text: my_data.to_string(),     // Tell vessel what the file content is
        }]
    }
}

// Export the plugin
export!(MyContentPlugin);
```

Compile it to WASM:
```bash
rustup target add wasm32-wasip2
cargo build -p my_content_mod --target wasm32-wasip2
```

### 2. Run the Generator (Host side)

Once you have the compiled `.wasm` file, use the `vessel` CLI tool to run it and output the files:

```bash
cargo run -p vessel -- build \
  target/wasm32-wasip2/debug/my_content_mod.wasm \
  --output ./assets/generated
```

**Alternatively, use it as a library**: If you are writing your own build scripts, you can call the API directly:

```rust
use vessel::prelude::*;

fn main() -> anyhow::Result<()> {
    // One-liner: Load the WASM, run it, and save the generated files to the target directory
    build_component("path/to/mod.wasm", "assets/generated")?;
    Ok(())
}
```

## How to Build

### Prerequisites

* A recent stable version of Rust.
* The `wasm32-wasip2` target (required if you want to run the included examples/tests).

### Build Steps

```bash
# 1. Clone the repository
git clone https://github.com/Bli-AIk/souprune.git
cd souprune/crates/vessel

# 2. Run tests
cargo test

# 3. View CLI help
cargo run -- --help
```
